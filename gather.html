Where this shows up — cross-entropy loss

A language model emits log_softmax scores of shape [T, V] — one distribution over the vocabulary at each position. The loss needs the log-probability of the gold next token at every position. That is a gather along the vocab axis (dim=-1): the target id picks the column, the position (row) stays pinned — exactly the dim=1 picture above.

vocab  0=the  1=cat  2=sat  3=on  4=mat  ·  sequence “the cat sat on mat”

logits   = model(tokens)                  # [T, V]
logprobs = F.log_softmax(logits, dim=-1)  # [T, V]
nll  = -logprobs.gather(-1, targets[:, None]).squeeze(-1)   # [T]
loss = nll.mean()                         # scalar — this is what F.cross_entropy does

…and elsewhere in an LLM stack

  • scatter — Mixture-of-Experts routing. Dispatch each token's hidden state into its chosen expert's buffer, then scatter_add the expert outputs back to the original token positions.
  • gather — beam search / KV-cache reorder. Each decode step, reorder the cache by the surviving beam indices: cache.gather(0, beam_idx).
  • scatter — one-hot & label smoothing. scatter_ a 1.0 (or a smoothed value) into the target column to build soft labels.
  • gather — top-k sampling / retrieval. Pull logits or embeddings at selected indices (top-k decoding, RAG document gather).
