Derivation

  1. A ring all-gather runs N−1 steps.
  2. Each step, every GPU sends exactly one shard (S bytes) to its right neighbor and receives one from its left.
  3. So each GPU sends, over the whole op,

  4. Rewriting with S = D/N:
    N−1⁄N · D  →  ≈ D as N grows
    i.e. the per-GPU volume is ~independent of N — ring all-gather is bandwidth-optimal.
  5. All N links run in parallel, so wall-clock time is set by the per-GPU volume, not the system total:
